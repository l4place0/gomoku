# v4-stabilize 训练结论

## 目标
从 v3 R3 (61.5%) 出发，修复评估噪声问题，稳定提升。

## 实际执行
- 计划 8 轮，实际完成 7 轮（R4 用户叫停）
- R1-R3 使用主线 checkpoint + v3 自博弈数据（不匹配），模型持续恶化
- R5 起对齐 checkpoint 与自博弈模型，胜率回升
- R6 阶段切换退化，R7-R8 稳定提升

## 结果

| 轮次 | 胜率 | 晋升 | 说明 |
|------|------|------|------|
| R1 | 43.0% | ✗ | checkpoint 与数据不匹配 |
| R2 | 40.4% | ✗ | 同上 |
| R3 | 22.0% | ✗ | 同上 |
| R4 | — | — | 停止 |
| R5 | 64.6% | ✓ | 对齐后首轮 |
| R6 | 33.3% | ✗ | 阶段切换退化 |
| R7 | 73.0% | ✓ | 适应后回升 |
| R8 | 81.0% | ✓ | 最佳 |

**最佳模型**：`6f156d9635c1`（81% 胜率）

## 修复项
1. Checkpoint NaN 自动清除（running_metrics 污染）
2. 哈希碰撞修复（file_hash + timestamp 混合）
3. FP16 默认关闭，改为可选参数
4. 40+ 训练参数暴露给 Agent

## 经验
1. Checkpoint 脏数据比 NaN 权重更隐蔽 — 需检查所有组件
2. 训练数据与模型必须同源 — 否则比两者都差
3. 阶段切换需要渐进 — 突然改变参数会退化
4. 100 局 PK 比 30 局可靠 — 置信区间从 ±15% 降到 ±8%
