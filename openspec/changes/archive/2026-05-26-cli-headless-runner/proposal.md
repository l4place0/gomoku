## Why

在当前的五子棋博弈引擎项目中，所有的 AI 对弈测试、参数调整和规则评估均需要拉起基于 Pygame 的图形界面 (`game.py`)。这带来了两个明显的限制：
1. **测试效率低下**：GUI 渲染和主事件循环帧率限制（即使在自对弈模式下）极大地拖慢了 AI 搜索推理（尤其是 KataGomo MCTS 神经网络）的速度。
2. **无法自动化集成**：在无图形界面的服务器或后台流水线中，无法拉起 Pygame 窗口，导致无法进行自动化的新旧模型 Elo 对决评估、自我对局数据跑谱收集、以及重新训练反馈的自动化闭环。

因此，现在急需开发一套纯 CLI 架构的“无头模式”对局运行器 (`headless_runner.py`)，它不依赖 Pygame 渲染，能以最大速度直接调用博弈 DLL 与多进程 AI Worker，并在对局结束时输出格式化的 JSON 对局日志与胜率统计。

## What Changes

1. **新增纯 CLI 无头对局运行器**：新建 `headless_runner.py` 脚本，作为一个完全独立于 Pygame GUI 的 CLI 工具。它能够独立加载指定的黑白双模型，并控制博弈 DLL 与后台多进程 Worker 进行极速自我对弈。
2. **标准化对局结果输出**：每一局结束以及整场对局赛段（Match）结束时，自动将落子历史、双方胜率、AI 决策路径（如三手交换和五手打点评估）以结构化的 JSON 文件格式输出，方便外部自动化脚本和训练流水线直接读取解析。
3. **保留并发跑谱扩展性**：在代码设计上将单场对局逻辑高度封装，并在参数层保留并发度配置（当前本机算力下默认限制并发对弈数为 1），为后续的集群并行跑谱测试埋下优雅接口。

## Capabilities

### New Capabilities
- `cli-headless-runner`: 提供了命令行无头对弈功能，支持配置黑白双模型路径、规则判定、对局盘数限制，并自动输出格式化的对局数据 JSON 报告。

## Impact

- **新增文件**：`headless_runner.py` 作为新的命令行入口。
- **依赖调用**：该运行器会直接载入 `GameEngine.dll`，并作为主进程通过 stdin/stdout 与 `ai_worker.py` 子进程（若配置了独立的白方模型）进行 IPC 管道通信。
- **环境要求**：支持在无图形界面的命令行或 Linux/Windows 后台纯 CLI 方式直接运行（只需 Python 基础环境，无需显式 Pygame 显示器支持）。
