## Why

当前系统的 Pygame 界面在游戏启动加载 KataGomo 神经网络模型、AI 评估三手交换决策、以及 AI 进行五手 N 打候选点计算时，由于计算线程的同步阻塞，导致主界面无任何视觉反馈与文字提示，造成假死感。同时，当游戏结束判定胜负时，“胜利标识”文本由于采用全屏宽度居中，容易被右下角的“悔棋”按钮发生视觉遮挡，影响对弈体验。

## What Changes

- **开局加载提示**：启动时在后台加载大模型与 CUDA 组件期间，展示专门的 UI 渲染页面及载入进度提示。
- **三手交换评估提示**：AI 执白评估三手交换抉择时，展示带有半透明锁定遮罩的“三手交换评估中”锁定弹窗。
- **五手 N 打 AI 运算提示**：AI 执黑计算并筛选 $N$ 个候选平衡点，以及 AI 执白评估保留落子时，均展示带有半透明锁定遮罩的“五手 N 打评估中”弹窗。
- **胜利标识布局避让**：将胜利标识提示的居中基准从窗口宽度 `WIDTH` 改为棋盘宽度 `BOARD_AREA_W`，完全避开右下角“悔棋”按钮的固定范围。

### Modified Capabilities

- `engineering`: 优化 UI 界面布局，增加各种非人类可执行对弈环节（如初始化加载、交换评估、五手 N 打评估）的 UI 视觉锁定与提示规范。

## Impact

- **game.py (Python UI & Main Loop)**：
  - 重构游戏启动初始化流程，在创建窗口后先渲染加载状态，再加载权重与 KataGomo 模型。
  - 在三手交换逻辑分支（第 3 手）与五手 N 打逻辑分支（第 4 手）的 AI 评估函数调用前，触发特定蒙版遮罩的绘制与刷新。
  - 修改 `winner` 状态渲染时的 blit 横坐标计算公式，由 `WIDTH` 居中调整为 `BOARD_AREA_W` 居中。
